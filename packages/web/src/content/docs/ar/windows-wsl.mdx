رغم أن Ottili Coder يمكن تشغيله مباشرة على Windows، نوصي باستخدام [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) للحصول على أفضل تجربة. يوفر WSL بيئة Linux تعمل بسلاسة مع ميزات Ottili Coder.

:::tip[لماذا WSL؟]
يوفر WSL أداء أفضل لنظام الملفات، ودعمًا كاملًا لـ terminal، وتوافقًا مع أدوات التطوير التي يعتمد عليها Ottili Coder.
:::

---

## الإعداد

<Steps>

1. **ثبّت WSL**

   إذا لم تقم بذلك بعد، [ثبّت WSL](https://learn.microsoft.com/en-us/windows/wsl/install) باستخدام دليل Microsoft الرسمي.

2. **ثبّت Ottili Coder داخل WSL**

   بعد إعداد WSL، افتح terminal داخل WSL وثبّت Ottili Coder باستخدام إحدى [طرق التثبيت](/docs/).

   ```bash
   curl -fsSL https://ottili.one/coder/install | bash
   ```

3. **استخدم Ottili Coder من WSL**

   انتقل إلى مجلد مشروعك (يمكنك الوصول لملفات Windows عبر `/mnt/c/` و`/mnt/d/` وغيرها) ثم شغّل Ottili Coder.

   ```bash
   cd /mnt/c/Users/YourName/project
   ottili-coder
   ```

</Steps>

---

## تطبيق سطح المكتب + خادم WSL

إذا كنت تفضّل تطبيق Ottili Coder لسطح المكتب لكن تريد تشغيل الخادم داخل WSL:

1. **ابدأ الخادم داخل WSL** مع `--hostname 0.0.0.0` للسماح بالاتصالات الخارجية:

   ```bash
   ottili-coder serve --hostname 0.0.0.0 --port 4096
   ```

2. **اربط تطبيق سطح المكتب** على `http://localhost:4096`

:::note
إذا لم يعمل `localhost` في إعدادك، اتصل باستخدام عنوان IP الخاص بـ WSL بدلًا منه (من داخل WSL: `hostname -I`) واستخدم `http://<wsl-ip>:4096`.
:::

:::caution
عند استخدام `--hostname 0.0.0.0`، اضبط `OTTILI_CODER_SERVER_PASSWORD` لحماية الخادم.

```bash
OTTILI_CODER_SERVER_PASSWORD=your-password ottili-coder serve --hostname 0.0.0.0
```

:::

## عميل الويب + WSL

لأفضل تجربة ويب على Windows:

1. **شغّل `ottili-coder web` من terminal داخل WSL** بدلًا من PowerShell:

   ```bash
   ottili-coder web --hostname 0.0.0.0
   ```

2. **افتحه من متصفح Windows** عبر `http://localhost:<port>` (يعرض Ottili Coder الرابط)

تشغيل `ottili-coder web` من WSL يضمن وصولًا صحيحًا لنظام الملفات وتكاملًا أفضل مع terminal، مع بقائه متاحًا من متصفح Windows.

---

## الوصول إلى ملفات Windows

يمكن لـ WSL الوصول إلى جميع ملفات Windows عبر مجلد `/mnt/`:

- قرص `C:` → `/mnt/c/`
- قرص `D:` → `/mnt/d/`
- وهكذا...

مثال:

```bash
cd /mnt/c/Users/YourName/Documents/project
ottili-coder
```

:::tip
لأفضل سلاسة، يمكنك استنساخ/نسخ المستودع إلى نظام ملفات WSL (مثل `~/code/`) وتشغيل Ottili Coder من هناك.
:::

---

## نصائح

- شغّل Ottili Coder داخل WSL للمشاريع المخزنة على أقراص Windows حتى يكون الوصول للملفات سلسًا
- استخدم [إضافة WSL في VS Code](https://code.visualstudio.com/docs/remote/wsl) مع Ottili Coder لسير عمل تطوير متكامل
- إعدادات Ottili Coder وجلساته تُخزَّن داخل بيئة WSL في `~/.local/share/ottili-coder/`
